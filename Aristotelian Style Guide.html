Style Guide

All tokens, components, and usage rules for the Aristotelian Character Forge. Light mode by default; dark mode via the toggle. Built on two typefaces, one border colour, and a strict no-gradient rule.

04 — Components

Buttons

Serif italic, 4px radius. Hover lifts translateY(-3px) with a dark drop-shadow. No coloured glows, no breathing animation. Use .btn-block for full-width (bottom-bar context only).

.btn .btn-primary
.btn-primary .btn-block
:disabled
<button class="btn btn-primary">Start building free</button> <button class="btn btn-primary btn-block">Forge Character</button> <!-- full width --> <button class="btn btn-primary" disabled>Disabled</button>
Use for: the single primary action on a screen — Create, Forge, Submit, Sign in. One per view.
✕ Don't — use for navigation, cancel, or secondary actions.
.btn .btn-ghost
.btn-ghost .btn-block
<button class="btn btn-ghost">See how it works</button> <button class="btn btn-ghost btn-block">Cancel</button>
Use for: secondary actions alongside a primary — Cancel, Back, "See how it works". Always paired with a primary.
.btn-sage
.btn-destroy
.btn-delete-world

.btn-sage — feature unlock / AI actions. .btn-destroy — destructive confirm (always inside a confirm flow). .btn-delete-world — text-only link style for low-priority destructive actions.

04 — Components

Bottom Bar

Fixed to viewport bottom. Contains the primary action for the current screen. Buttons here always use .btn-block.

<div class="bottom-bar"> <div class="bottom-bar-inner"> <button class="btn btn-primary btn-block">+ New World</button> <!-- optional secondary: --> <button class="btn btn-ghost btn-block">Cancel</button> </div> </div>
Max two buttons in a bottom bar: primary on top, ghost Cancel below. Never more.
✕ Don't — use full-width buttons outside of bottom-bar or modal contexts.

05 — Principles

Design Rules

No gradients

Background fills are flat colours only. No linear-gradient, no radial-gradient. Depth is achieved with border, shadow, and layered surface colours.

Amber is precious

Gold/amber appears in two places: the small brand dot in the nav mark, and all quote text (.t-quote, .char-quote, .card-quote). Everywhere else it is off-limits. Don't use it for labels, badges, or UI chrome.

No pill shapes

Interactive elements (buttons, tags, inputs) use --r-sm (4px). Pill radii (50px) are forbidden. Cards use --r-lg; modals use --r-xl.

No animations on buttons

Buttons do not breathe, pulse, or glow. The only motion is translateY(-3px) on hover and scale(.97) on press. Shadows are dark, never coloured.

Light by default

The app ships in light mode. Dark mode is opt-in via a nav toggle and is persisted in localStorage. The body class (theme-dark) drives all dark overrides.

One primary per screen

Only one .btn-primary per view. All other actions are ghost, text-only, or icon buttons. This keeps the visual hierarchy unambiguous.
